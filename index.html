Dijkstra's Algorithm

Finds the shortest weighted path in a graph using a priority queue. Used here to route emergency vehicles through city road networks with minimal travel distance.

🎒
0/1 Knapsack DP

Dynamic Programming approach that maximizes total value of selected items without exceeding weight capacity. Used to allocate city emergency resources optimally.

📅
Greedy Job Scheduling

Sorts city services by profit and greedily assigns them to time slots before their deadlines — guaranteeing maximum revenue from available service windows.

Complexity Analysis

Formal Big-O analysis of each algorithm — use this table during viva voce to justify the design decisions and computational trade-offs of the system.

Module Algorithm Paradigm Time Complexity Space Complexity Justification
Emergency Routing Dijkstra's Algorithm Graph O((V + E) log V) O(V + E) Priority queue guarantees minimum-distance node extraction in O(log V). Optimal for sparse city road networks.
Resource Allocation 0/1 Knapsack Dynamic Prog. O(n × W) O(n × W) Builds a 2D DP table. Guarantees globally optimal selection — greedy would fail here due to non-fractional items.
Service Scheduling Job Sequencing with Deadlines Greedy O(n log n + n·d) O(d) Profit-based greedy sort + slot assignment. Provably optimal for maximizing profit in single-machine scheduling.
